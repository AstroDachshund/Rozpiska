# M4 S4.2 — PlanBuilderTree Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A working tree editor for plan templates — trainer can create a template, then add/rename/delete weeks, days, and sections (warmup/main/cooldown) entirely with the keyboard, on top of the S4.1 data layer. No exercises/sets UI yet (S4.3).

**Architecture:** `lib/plan-builder/templates.ts` + `template-actions.ts` add the missing template-level CRUD (list/create — S4.1 only covered the tree below a root). `components/plan-builder/*` is a bottom-up stack of client components (`SectionRow` → `DayNode` → `WeekNode` → `PlanBuilderTree`) that render `usePlanTree(context)` and call the S4.1 mutation hooks directly — no new mutation code. Two routes: `(trainer)/templates` (list + create) and `(trainer)/templates/[id]` (the builder).

**Tech Stack:** Next.js 15 (RSC list/detail pages + client tree components) · `@tanstack/react-query` v5 (already wired) · `sonner` (new — mutation-error toasts) · Zod · Vitest.

## Global Constraints

- **No migration, no `gen types`** — schema unchanged (`plan_templates`, `plan_weeks/days/sections` already exist from M1).
- **Zero new mutation code.** Every create/rename/delete in the tree UI calls an existing S4.1 hook (`use{Create,Update,Delete}{Week,Day,Section}` from `lib/plan-builder/mutations/*`). This session is UI wiring only.
- **`PlanBuilderTree` takes exactly one prop: `context: PlanContext`** (the same discriminated union from S4.1 — `{kind:'template', templateId}` or `{kind:'assigned', assignedPlanId}`). Do not add a separate `rootId` prop even though design-system.md §5.5's literal table shows `rootId`/`context` — `PlanContext` already carries both, and the component is only ever agnostic through this one prop.
- **Section model: exactly one section per type, per day.** `SECTION_TYPES = ['warmup', 'main', 'cooldown'] as const` and `SECTION_TYPE_LABELS` are defined once, in `components/plan-builder/section-row.tsx`, and imported everywhere else that needs them (day-node's "missing type" buttons). Never redeclare this list.
- **Display order of sections within a day is always warmup → main → cooldown**, regardless of `position` or creation order — sort by `SECTION_TYPES.indexOf(section_type)` at render time in `day-node.tsx`. This is a display-only sort; it never writes to the `position` column.
- **Node creation is zero-form** (per approved design): "+ Dodaj tydzień" auto-numbers (`max(week_number) + 1`); "+ Dodaj dzień" creates with placeholder name `Dzień {n}` and immediately opens inline-rename (`autoFocus`). Neither waits on a dialog or extra confirmation step.
- **Delete uses a native `confirm()`** before calling the delete mutation — no `AlertDialog`, no custom modal. One confirmation string per node level (week/day/section), each naming what will be destroyed, e.g. `` `Usunąć dzień „${day.name}” wraz z zawartością?` ``.
- **Mutation errors surface via `sonner` toast** (`toast.error('Nie udało się ...')`) passed as the `onError` callback on each `.mutate(...)` call — never swallowed silently.
- **Every "add" button is disabled while its own create mutation is in flight** (`disabled={createX.isPending}`): the S4.1 final review flagged that two rapid clicks read the same stale `queryClient` cache before either mutation's `onSettled` refetch lands, computing the same `lastPosition`/`week_number` twice and creating two siblings with colliding values. This is a targeted per-button guard, not optimistic UI or cross-tree serialization — it stops exactly the double-click case. Applies to: the section "+" buttons (Task 7), "+ Dodaj dzień" (Task 8), "+ Dodaj tydzień" (Task 9).
- **`sonner` is hand-wired without the `next-themes` dependency.** The stock `npx shadcn add sonner` output imports `useTheme` from `next-themes`, which this app doesn't use (themes are per-route-group CSS classes, not a runtime toggle — see CLAUDE.md UI rules). Task 1 installs the bare `sonner` package and hand-writes `components/ui/sonner.tsx` with `theme="light"` fixed, following the exact style-token wiring the generator would produce otherwise (`--normal-bg`/`--normal-text`/`--normal-border` mapped to `--popover`/`--popover-foreground`/`--border`). `<Toaster />` is mounted in `app/(trainer)/layout.tsx` only (not the root layout) since every toast this session originates from trainer screens, which are always light-themed — revisit when client-side (dark) toasts are needed (M7).
- **No component-level tests.** `vitest.config.ts` runs with `environment: 'node'` (no jsdom) — same constraint as S4.1. Vitest only covers the new `templateInputSchema` (pure) and the `lib/auth/routes.ts` addition (pure). Everything else is verified live in the browser (Task 11).
- **`lib/auth/routes.ts` gets `/templates` added to `TRAINER_PREFIXES`** in the same task that creates the first `/templates` route — this is the exact gap (F6) that shipped unregistered in S3.1 and was caught only in final review; do not repeat it.
- **Before every commit**: `npm run typecheck` → `npm run lint` → `npm run format:check` → `npm run test`. `format:check` is mandatory (CI fails on it).
- Branch `feat/m4-builder-tree`, based on current `main` (S4.1 merged). One PR.

```typescript
// Serwerowe odczyty szablonów (RSC). RLS zawęża do trenera (plan_templates_trainer_all).
import { createClient } from '@/lib/supabase/server';
import type { Database } from '@/lib/supabase/types';

export type TemplateRow = Pick<
  Database['public']['Tables']['plan_templates']['Row'],
  'id' | 'name' | 'created_at'
>;

export async function listTemplates(): Promise<TemplateRow[]> {
  const supabase = await createClient();
  const { data, error } = await supabase
    .from('plan_templates')
    .select('id, name, created_at')
    .is('archived_at', null)
    .order('created_at', { ascending: false });
  if (error) throw error;
  return data;
}

export async function getTemplate(id: string): Promise<TemplateRow | null> {
  const supabase = await createClient();
  const { data, error } = await supabase
    .from('plan_templates')
    .select('id, name, created_at')
    .eq('id', id)
    .maybeSingle();
  if (error) throw error;
  return data;
}
```

**Files:**

**Interfaces:**

- Produces: `TemplateFormState`, `createTemplateAction(prevState, formData)`.

- [ ] **Step 1: Create `lib/plan-builder/template-actions.ts`**

```typescript
'use server';
import { redirect } from 'next/navigation';
import { createClient } from '@/lib/supabase/server';
import { templateInputSchema } from '@/lib/plan-builder/schemas';

export type TemplateFormState = { error?: string };

export async function createTemplateAction(
  _prev: TemplateFormState,
  formData: FormData
): Promise<TemplateFormState> {
  const parsed = templateInputSchema.safeParse({ name: formData.get('name') });
  if (!parsed.success) return { error: parsed.error.issues[0]!.message };

  const supabase = await createClient();
  const {
    data: { user },
  } = await supabase.auth.getUser();
  if (!user) return { error: 'Musisz być zalogowany.' };

  const { data: created, error } = await supabase
    .from('plan_templates')
    .insert({ trainer_id: user.id, name: parsed.data.name })
    .select('id')
    .single();
  if (error || !created) return { error: 'Nie udało się utworzyć szablonu.' };

  redirect(`/templates/${created.id}`);
}
```

```tsx
'use client';
import { useState } from 'react';

type Props = {
  value: string;
  onCommit: (value: string) => void;
  ariaLabel: string;
  required?: boolean;
  placeholder?: string;
  className?: string;
  autoFocus?: boolean;
};

const INPUT_CLASS =
  'h-8 w-full rounded-md border border-input bg-transparent px-2 text-sm outline-none focus-visible:ring-2 focus-visible:ring-ring';

/**
 * Klik/focus -> input. Enter/blur zatwierdza, Esc cofa do poprzedniej wartości.
 * `required=true` blokuje zatwierdzenie pustej wartości (zamiast tego wraca do poprzedniej).
 * `autoFocus=true` startuje od razu w trybie edycji — używane przy tworzeniu węzła
 * z nazwą-placeholderem (np. nowy dzień "Dzień N").
 */
export function InlineEditableText({
  value,
  onCommit,
  ariaLabel,
  required = false,
  placeholder = 'Dodaj notatkę',
  className,
  autoFocus = false,
}: Props) {
  const [editing, setEditing] = useState(autoFocus);
  const [draft, setDraft] = useState(value);

  function startEditing() {
    setDraft(value);
    setEditing(true);
  }

  function commit() {
    const trimmed = draft.trim();
    if (required && trimmed.length === 0) {
      setEditing(false);
      return;
    }
    setEditing(false);
    if (trimmed !== value) onCommit(trimmed);
  }

  function cancel() {
    setDraft(value);
    setEditing(false);
  }

  if (editing) {
    return (
      <input
        autoFocus
        aria-label={ariaLabel}
        value={draft}
        onChange={(e) => setDraft(e.target.value)}
        onBlur={commit}
        onKeyDown={(e) => {
          if (e.key === 'Enter') {
            e.preventDefault();
            commit();
          } else if (e.key === 'Escape') {
            e.preventDefault();
            cancel();
          }
        }}
        className={className ?? INPUT_CLASS}
      />
    );
  }

  return (
    <button type="button" onClick={startEditing} aria-label={ariaLabel} className={className}>
      {value || `+ ${placeholder}`}
    </button>
  );
}
```

```tsx
'use client';
import { Trash2 } from 'lucide-react';
import { toast } from 'sonner';
import { Button } from '@/components/ui/button';
import { useDeleteSection } from '@/lib/plan-builder/mutations/use-sections';
import type { PlanContext, PlanSection } from '@/lib/plan-builder/types';

export const SECTION_TYPES = ['warmup', 'main', 'cooldown'] as const;

export const SECTION_TYPE_LABELS: Record<PlanSection['section_type'], string> = {
  warmup: 'Rozgrzewka',
  main: 'Główna',
  cooldown: 'Cooldown',
};

export function SectionRow({ context, section }: { context: PlanContext; section: PlanSection }) {
  const deleteSection = useDeleteSection(context);

  function handleDelete() {
    const label = SECTION_TYPE_LABELS[section.section_type];
    if (!confirm(`Usunąć sekcję „${label}” wraz z zawartością?`)) return;
    deleteSection.mutate(section.id, {
      onError: () => toast.error('Nie udało się usunąć sekcji.'),
    });
  }

  return (
    <li className="flex items-center justify-between gap-2 rounded-md border border-border bg-muted px-3 py-2 text-sm">
      <span className="font-medium text-foreground">
        {SECTION_TYPE_LABELS[section.section_type]}
      </span>
      <Button
        type="button"
        variant="ghost"
        size="icon-sm"
        aria-label={`Usuń sekcję ${SECTION_TYPE_LABELS[section.section_type]}`}
        onClick={handleDelete}
      >
        <Trash2 className="size-4" strokeWidth={1.75} />
      </Button>
    </li>
  );
}
```

```tsx
  // Wyświetlanie zawsze rozgrzewka -> główna -> cooldown, niezależnie od position/kolejności
  // utworzenia. To sortowanie wyłącznie prezentacyjne — nie zapisuje position.
  const sortedSections = [...day.sections].sort(
    (a, b) => SECTION_TYPES.indexOf(a.section_type) - SECTION_TYPES.indexOf(b.section_type)
  );
  const missingTypes = SECTION_TYPES.filter(
    (t) => !day.sections.some((s) => s.section_type === t)
  );

  function handleDelete() {
    if (!confirm(`Usunąć dzień „${day.name}” wraz z zawartością?`)) return;
    deleteDay.mutate(day.id, {
      onError: () => toast.error('Nie udało się usunąć dnia.'),
    });
  }

  return (
    <li className="rounded-lg border border-border bg-card p-3">
      <div className="flex items-center justify-between gap-2">
        <InlineEditableText
          value={day.name}
          ariaLabel={`Nazwa dnia: ${day.name}`}
          required
          autoFocus={autoFocusName}
          onCommit={(name) =>
            updateDay.mutate(
              { id: day.id, name },
              { onError: () => toast.error('Nie udało się zapisać nazwy dnia.') }
            )
          }
          className="text-left text-base font-medium text-foreground hover:underline"
        />
        <Button
          type="button"
          variant="ghost"
          size="icon-sm"
          aria-label={`Usuń dzień ${day.name}`}
          onClick={handleDelete}
        >
          <Trash2 className="size-4" strokeWidth={1.75} />
        </Button>
      </div>

      <ul className="mt-3 space-y-2">
        {sortedSections.map((section) => (
          <SectionRow key={section.id} context={context} section={section} />
        ))}
      </ul>

      {missingTypes.length > 0 && (
        <div className="mt-3 flex flex-wrap gap-2">
          {missingTypes.map((t) => (
            <Button
              key={t}
              type="button"
              variant="outline"
              size="sm"
              disabled={createSection.isPending}
              onClick={() =>
                createSection.mutate(
                  { day_id: day.id, section_type: t },
                  { onError: () => toast.error('Nie udało się dodać sekcji.') }
                )
              }
            >
              <Plus className="size-4" strokeWidth={1.75} /> {SECTION_TYPE_LABELS[t]}
            </Button>
          ))}
        </div>
      )}
    </li>
  );
}
```

```tsx
import { Plus } from 'lucide-react';
import { toast } from 'sonner';
import { Button } from '@/components/ui/button';
import { WeekNode } from '@/components/plan-builder/week-node';
import { usePlanTree } from '@/lib/plan-builder/queries';
import { useCreateWeek } from '@/lib/plan-builder/mutations/use-weeks';
import type { PlanContext } from '@/lib/plan-builder/types';

export function PlanBuilderTree({ context }: { context: PlanContext }) {
  const { data: weeks, isPending } = usePlanTree(context);
  const createWeek = useCreateWeek(context);

  function handleAddWeek() {
    const nextNumber =
      weeks && weeks.length > 0 ? Math.max(...weeks.map((w) => w.week_number)) + 1 : 1;
    createWeek.mutate(
      { week_number: nextNumber },
      { onError: () => toast.error('Nie udało się dodać tygodnia.') }
    );
  }

  if (isPending) {
    return <p className="mt-8 text-sm text-muted-foreground">Wczytywanie planu…</p>;
  }

  return (
    <div>
      {weeks && weeks.length === 0 && (
        <p className="mt-4 text-sm text-muted-foreground">
          Ten szablon nie ma jeszcze żadnego tygodnia.
        </p>
      )}

      <ul className="mt-4 space-y-4">
        {weeks?.map((week) => <WeekNode key={week.id} context={context} week={week} />)}
      </ul>

      <Button
        type="button"
        variant="outline"
        className="mt-4"
        disabled={createWeek.isPending}
        onClick={handleAddWeek}
      >
        <Plus className="size-4" strokeWidth={1.75} /> Dodaj tydzień
      </Button>
```
