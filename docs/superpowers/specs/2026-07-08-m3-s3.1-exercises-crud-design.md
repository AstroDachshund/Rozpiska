# M3 S3.1 — Bank ćwiczeń (CRUD) — design (spec)

Data: 2026-07-08 · Status: zatwierdzony w brainstormingu (Piotr) · Milestone: **M3**, sesja **S3.1** (`feat/m3-exercises-crud`)

Kanon zakresu: `docs/plan-claude-code.md §3`. Ten spec dokłada rozstrzygnięcia: mechanizm danych, model tagów, kształt Zod, prezentację formularza, seed. Rozbieżność spec↔plan rozstrzyga plan.

## 1. Rozstrzygnięcia (z brainstormingu)

- **Bez migracji.** Tabele `exercises`, `exercise_tags`, `exercise_tag_links` + RLS istnieją z M1 (`20260705193045_exercises_tags.sql`). S3.1 to wyłącznie warstwa aplikacji. Testy izolacji tenantów z M1 (`supabase/tests/tenant-isolation.test.ts`) już pokrywają widoczność ćwiczeń → **brak nowych testów RLS**.
- **Mechanizm danych: RSC + Server Actions** (spójnie z auth/invites). Lista `(trainer)/exercises` to Server Component czytający przez klienta SSR (RLS zawęża do trenera); mutacje przez Server Actions z walidacją Zod i `revalidatePath('/exercises')`. **Bez TanStack Query** — provider wchodzi dopiero w M4 (kreator) / M6 (cache klienta).
- **Tagi: seed + przypisywanie + inline create.** Seed standardowego polskiego zestawu tagów per trener; formularz ćwiczenia przypisuje z istniejących (chipy-toggle grupowane po kategorii) oraz pozwala utworzyć nowy tag inline (nazwa + kategoria). `createTagAction` obsługuje konflikt `unique (trainer_id, category, name)` zwracając istniejący tag (idempotentnie).
- **Soft delete przez `archived_at`** (nigdy hard delete — `set_logs` w M7 będą się odwoływać do ćwiczeń). Zarchiwizowane w osobnym widoku listy (`?archived=1`), z akcją „Przywróć".
- **Formularz na dedykowanych trasach** `(trainer)/exercises/new` i `(trainer)/exercises/[id]/edit` (bez zależności od komponentu Dialog). Edycja: RSC ładuje ćwiczenie + jego tagi.

## 2. Zod (`lib/exercises/schemas.ts`) — jedno źródło (formularz + serwer)

- `tagCategorySchema = z.enum(['muscle_group','equipment','movement_pattern'])`.
- `exerciseInputSchema`:
  - `name`: `z.string().trim().min(1, 'Podaj nazwę ćwiczenia.').max(120, …)`.
  - `technique_note`: `z.string().trim().max(2000, …).optional()` (pusty → `undefined`/`null`).
  - `youtube_url`: opcjonalny; pusty string → `undefined`; w przeciwnym razie `z.string().url(…)` + walidacja hosta (`youtube.com` / `youtu.be`) z polskim komunikatem. Luźna — akceptuje warianty (`www.`, `m.`, `youtu.be`, `/watch`, `/shorts`).
  - `tag_ids`: `z.array(z.string().uuid()).default([])`.
- `tagInputSchema`: `{ category: tagCategorySchema, name: z.string().trim().min(1).max(60) }`.
- Testy Vitest (`lib/exercises/schemas.test.ts`): name wymagane/trim; za długie name/note odrzucone; youtube_url — pusty ok, poprawny youtube ok, nie-URL odrzucony, nie-youtube URL odrzucony; tag_ids nie-uuid odrzucone; kategoria spoza enuma odrzucona.

## 3. Server Actions (`lib/exercises/actions.ts`)

Wszystkie: `'use server'`, walidacja Zod, klient SSR (RLS wymusza własność trenera — `exercises_trainer_all` z with-check `auth.uid() = trainer_id`), na końcu `revalidatePath('/exercises')` (i redirect dla create/update).

- `createExerciseAction(prev, formData)` — insert `exercises` (`trainer_id = auth.uid()`, `name`, `technique_note`, `youtube_url`), potem batch-insert `exercise_tag_links` dla `tag_ids`. Zapisy sekwencyjne w akcji (nie RPC — niskie ryzyko; częściowa awaria = ćwiczenie bez części linków, naprawialne edycją). Redirect na `/exercises`.
- `updateExerciseAction(prev, formData)` — update pól; **rekoncyliacja linków**: pobierz istniejące `tag_ids` ćwiczenia, usuń odjęte, wstaw dodane (diff), zamiast delete-all+reinsert. Redirect na `/exercises`.
- `archiveExerciseAction(id)` / `restoreExerciseAction(id)` — set/clear `archived_at = now()`/`null`. `revalidatePath`.
- `createTagAction(prev, formData)` — insert `exercise_tags(trainer_id, category, name)`; przy konflikcie `unique` — select i zwróć istniejący (idempotentnie). Zwraca `{ tag }` lub `{ error }`.
- Typ własności: `trainer_id` zawsze z `auth.uid()` po stronie serwera, nigdy z formularza.

## 4. UI

- `app/(trainer)/exercises/page.tsx` (RSC): czyta `exercises` (aktywne albo `archived` wg `searchParams.archived`) z ich tagami (jedno zapytanie z zagnieżdżonym select `exercise_tag_links(exercise_tags(*))` albo dwa zapytania złączone w kodzie). Render: nagłówek + „Dodaj ćwiczenie" (link `/exercises/new`) + przełącznik Aktywne/Zarchiwizowane; lista wierszy: nazwa, `TagChip`-y (kategorie), link do wideo (ikona), akcje Edytuj / Archiwizuj (albo Przywróć). Pusty stan: „Nie masz jeszcze żadnych ćwiczeń. Dodaj pierwsze." Motyw trenera = jasny, desktop-first, gęstość krok 8 (design-system §3).
- `app/(trainer)/exercises/new/page.tsx` — renderuje `ExerciseForm` (kontekst create); ładuje listę tagów trenera do wyboru.
- `app/(trainer)/exercises/[id]/edit/page.tsx` (RSC) — ładuje ćwiczenie + jego `tag_ids` + listę tagów; renderuje `ExerciseForm` (kontekst edit). 404 jeśli nie należy do trenera (RLS zwróci brak).
- `components/exercise-form.tsx` (klient, `useActionState`): pola name / technique_note (Textarea) / youtube_url; sekcja tagów — chipy-toggle grupowane po kategorii (muscle_group / equipment / movement_pattern), zaznaczone = w `tag_ids` (hidden inputs lub stan + serializacja); inline „+ Nowy tag" (nazwa + wybór kategorii) wołający `createTagAction`, po sukcesie dodaje chip i zaznacza. Przyciski nazywają akcję („Zapisz ćwiczenie"). Focus/klawiatura zgodnie z design-system §5.
- `components/tag-chip.tsx` — chip `rounded-full` (radius 999), warianty: `selectable` (toggle, `aria-pressed`), statyczny (render na liście). Kolor kategorii z palety `plate-*`/muted (dekoracja; stan nie tylko kolorem — zaznaczenie też ringiem/ikoną). Widoczny focus `ring-2 ring-ring ring-offset-2`.
- `components/ui/textarea.tsx` — shadcn Textarea na tokenach (spójnie z Input).

## 5. Seed (realny polski content)

Rozszerzyć seed dev o dane tenantowe trenera `trener@rozpiska.local` (przez Admin API / service_role — jak `supabase/seed/dev-users.mjs`). Osobny krok/skrypt `supabase/seed/exercises.mjs` (lub dopisany), idempotentny (upsert po naturalnych kluczach):
- **Tagi** (per trener): muscle_group: Klatka, Plecy, Nogi, Barki, Biceps, Triceps, Brzuch, Pośladki; equipment: Sztanga, Hantle, Maszyna, Wyciąg, Masa ciała, Kettlebell; movement_pattern: Pchanie poziome, Pchanie pionowe, Ciągnięcie poziome, Ciągnięcie pionowe, Przysiad, Zawias biodrowy.
- **Ćwiczenia** (~8–10) z tagami + notatką techniczną + linkiem YouTube: np. „Przysiad ze sztangą", „Wyciskanie sztangi na ławce płaskiej", „Martwy ciąg", „Wiosłowanie sztangą w opadzie", „Podciąganie na drążku", „Wyciskanie hantli nad głowę", „Uginanie ramion ze sztangą", „Wykroki z hantlami". Nigdy lorem ipsum (CLAUDE.md working style).
